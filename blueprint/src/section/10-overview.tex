\section{Universal Composability}

Universal composability states security as an execution comparison.  A real
protocol is secure if every adversary attacking it can be matched by a
simulator in an ideal execution, so that no efficient environment can tell
which execution it is interacting with.  The environment is the observer and
test harness: it supplies inputs, interacts with the adversarial interface, and
finally outputs the bit whose distribution defines the security experiment.

This blueprint follows the Lean development rather than a paper-specific
notation.  The core model uses explicit machines, communication ports,
message-level instructions, a controller that serializes activations, and
ideal protocols generated from ideal functionalities by dummy parties.  The
case studies then instantiate this core with concrete functionalities and
protocol machines.
